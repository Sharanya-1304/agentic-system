Day 9: Reasoning

1. Chain of Thought (CoT)

Standard LLMs jump to conclusions. CoT forces them to show their work.

Standard Prompting

Input: Roger has 5 balls. He buys 2 cans of tennis balls. Each can has 3 balls. How many does he have?

Output: 11.

(Often wrong on complex math)

CoT Prompting

Output: Roger started with 5.
2 cans * 3 balls = 6 balls.
5 + 6 = 11.
Answer: 11.

# Implementation: Just add this to your system prompt
system_prompt = """
Answer the user's question. 
CRITICAL: Before answering, write out your step-by-step reasoning 
inside  tags.
"""

2. Tree of Thoughts (ToT)

For creative or complex tasks, one path isn't enough. ToT explores multiple possibilities and discards bad ones.

The Algorithm

  1. Decompose: Break problem into steps (e.g., Outline -> Draft -> Edit).
  2. Generate: Create 3 options for the current step.
  3. Evaluate: Score each option (0.0 to 1.0).
  4. Select: Keep the best, discard the rest.

3. Graph of Thoughts (GoT)

Trees branch out, but Graphs can merge back. Thoughts can combine.

Concept

Agent generates 3 distinct summaries of a document. Then, a 4th step aggregates the best parts of all 3 into a Master Summary.

Code Logic

# NetworkX-style logic
G.add_node("summary_1")
G.add_node("summary_2")
G.add_node("final_merge")
G.add_edge("summary_1", "final_merge")
G.add_edge("summary_2", "final_merge")

Applying Reasoning to Learning

How a "Tutor Agent" uses CoT to grade a student.

Internal Monologue (Invisible to Student)

Student Answer: "Photosynthesis is when plants eat dirt."

<thought>
1. Analyze accuracy: Incorrect. Plants perform synthesis, they don't eat dirt.
2. Identify misconception: Confusing nutrient absorption with energy production.
3. Plan response: Validate effort, correct the "dirt" part, explain light energy.
</thought>

Agent Output: "Not quite! Plants actually use sunlight to make food..."

The Self-Correction Loop

Code generation agents often fail on the first try. We need a loop.

The Architecture

while attempts < max_retries:
    code = generator.write_code(task)
    error = executor.run(code)
    
    if not error:
        return code # Success
    
    # The Critical Step
    plan = critic.analyze(code, error)
    generator.update_prompt(plan)

The Validation Workflow

  • Step 1 (Consistency): Does the Abstract match the Conclusion? (CoT check).
  • Step 2 (Citation Check): Extract references. Are they real? (Tool verification).
  • Step 3 (Novelty): Compare hypothesis against 100 recent papers (Vector Search).
