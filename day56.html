Day 5: Concepts

What is a Workflow?

A single agent is a worker. A Workflow is the assembly line.

Atomic vs. Composite

Atomic (Single Step)

An agent that does one thing perfectly.

# Single Task
summary = Summarizer.run(text)

Composite (Pipeline)

A chain of atomic agents passing data.

# Chained Workflow
data = search(query)
text = read(data)
summ = summarize(text)
email(summ)

Key Principle: Do not build "God Agents" that do everything. Build small specialists and chain them.

Data Flow & Context Handoff

The hardest part of automation is Context Propagation. How does Agent B know what Agent A found?

The "State" Object

We pass a shared dictionary (State) between agents.

state = {
  "original_query": "Check thesis.pdf for plagiarism",
  "extracted_text": "...",  # Added by PDF Agent
  "search_results": [...],  # Added by Search Agent
  "final_score": 0.85       # Added by Critic Agent
}

Use Case Build

Design: Plagiarism Checker

We will build this pipeline using a linear chain.

The Linear Flow Code

class PlagiarismPipeline:
    def run(self, pdf):
        # Step 1: Parse
        text = self.parser.extract(pdf)
        
        # Step 2: Research
        evidence = self.researcher.search(text)
        
        # Step 3: Evaluate
        report = self.judge.evaluate(text, evidence)
        
        return report
Day 6: Advanced

Beyond Linear Chains

Real world tasks are not always Step 1 -> Step 2. They require logic.

Conditional Branching

If X happens, do A. If Y happens, do B.

Example: If data is dirty -> Clean it. If clean -> Analyze it.

Parallel Execution

Run multiple agents at the same time (Fan-out) and aggregate results (Fan-in).

Example: Search Google, Bing, and Wikipedia simultaneously.

The Router Pattern

A "Gateway Agent" analyzes the user request and routes it to the correct specialist.

Why use a Router?

Specialized agents perform better than general ones. A "Math Agent" is better at calculation than a "Poetry Agent". The Router ensures the right tool for the job.

Use Case Build

Design: Data Analysis Agent

This workflow requires branching and loops.

Step 1: Load Data (CSV/Excel)
Step 2: Quality Check (Router Decision)
Branch A: Cleaner Agent (If missing values)
Branch B: Visualizer Agent (If clean)
Step 3: Final Report

Implementing Branching (Python)

Using langgraph or simple conditional logic.

def router(state):
    # Decide where to go based on data quality
    if state["missing_values"] > 0:
        return "cleaner_agent"
    else:
        return "visualizer_agent"

# Define the Workflow Graph
workflow = StateGraph(State)
workflow.add_node("router", router)
workflow.add_node("cleaner", cleaner_agent)
workflow.add_node("visualizer", visualizer_agent)

# Define Routing Logic
workflow.add_conditional_edges("router", {
    "cleaner_agent": "cleaner",
    "visualizer_agent": "visualizer"
})

Optimization Strategies

Pipelines can get expensive and slow. How do we fix this?

Caching (The Wallet Saver)

If Step 1 (Search) costs $0.10 and takes 5s, Cache it. If the user asks the same question, return the cached result instantly.

Model Cascade

Don't use GPT-4 for everything.
1. Use Llama-3-8B (Fast/Cheap) for routing/formatting.
2. Use GPT-4 (Smart/Expensive) only for complex reasoning.
