Day 3: Architecture

The Reasoning Loop

Simple scripts fail when tools break. Agentic Loops are designed to fail and recover.

The "Refinement" Pattern

Instead of giving up on error, the agent analyzes the error and adjusts its plan.

Goal: "Summarize URL"
Attempt 1: Scrape(URL) -> Error 403 (Forbidden)
Reflection: "The site blocked me. I should try Google Cache or Archive.org."
Attempt 2: Scrape(Archive_URL) -> Success

Day 4: Roles

Defining Roles

A "Role" is a combination of Behavior, Knowledge, and Constraints.

1. Specialization

Don't ask one agent to do everything.
Bad: "You are a helpful assistant."
Good: "You are a Senior Python Architect specializing in pandas optimization."

2. Constraints

Defining what the agent cannot do is as important as what it can do.
"Do not invent citations. If no data is found, state 'Unknown'."

Role Switcher

See how changing the System Prompt dramatically changes the output for the exact same query.

SYSTEM: You are an empathetic Tutor. Use analogies. No jargon.

User Query: "Explain how Neural Networks learn."

Agent Output:

Imagine a Neural Network like a team of musicians trying to play a new song. At first, they are noisy (random weights). But after every practice, the conductor tells them who was off-key (Backpropagation). Slowly, they tune their instruments until the song sounds perfect!

Project: Use Case 1

Learning Agent Architecture

Applying Day 3 concepts to build the "Learning Support" module.

Workflow Design

  • 1. Ingest: Student uploads a confusing PDF chapter.
  • 2. Analyze: Agent identifies key concepts and complexity level.
  • 3. Adapt (Reasoning):
    If student is beginner -> Generate Quiz + Glossary.
    If student is advanced -> Generate Thesis Scenarios.
  • 4. Output: Return personalized study material.
Self-Correction Mechanism: If the generated quiz is too hard (student scores < 50%), the agent autonomously regenerates a simplified version.
